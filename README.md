## README (current status)

### What this project does (so far)
This repo contains a **baseline SAT-based ATPG flow** for ISCAS-style gate-level Verilog benchmarks.

Given a benchmark and a **stuck-at fault** \((signal, sa0/sa1)\), we:
- Build a **good** and **faulty** copy of the circuit in CNF
- Force the faulty signal to 0 or 1 (fault injection)
- Tie good/faulty **primary inputs** to be identical
- Add a **miter** constraint to ensure good output \(\neq\) faulty output
- Use **MiniSat (PySAT Minisat22)** to solve and collect solver metrics

### Files added / updated
- `src/cnf_encoder.py`
  - Encodes gate-level logic into CNF
  - Supported gates: `NOT`, `AND` (n-input), `OR` (n-input), `NAND` (n-input), `NOR` (n-input), `XOR` (2-input)
  - Helpers for fault injection and output miter (currently uses the **first primary output**)

- `src/sat_atpg_demo.py`
  - Parses ISCAS-style Verilog (`input`, `output`, and gate instance lines)
  - Builds SAT instances for stuck-at faults
  - Runs ATPG over **all faults** in a circuit and returns metrics:
    - detected faults / total faults (coverage)
    - wall time, SAT time
    - average SAT decisions and conflicts per fault

- `src/benchmark_runner.py`
  - Runs ATPG + metrics for **all provided benchmarks**
  - Prints a clean terminal table
  - Writes results to `benchmark_metrics.csv`

- `benchmark_metrics.csv`
  - CSV output generated by `benchmark_runner.py` with per-benchmark metrics

### Benchmarks included
Located in `Benchmarks/`:
- `c432.v`, `c499.v`, `c880.v`, `c1355.v`, `c3540.v`, `c6288.v`, `s298.v`, `s344.v`

### How to run
From the repo root:

- **Run metrics on all benchmarks** (recommended):

```bash
python src/benchmark_runner.py
```

This will:
- print a table (coverage, time, avg decisions/conflicts)
- write `benchmark_metrics.csv`

- **Run a single benchmark**:
Edit the last line in `src/sat_atpg_demo.py` and run:

```bash
python src/sat_atpg_demo.py
```

### Notes / current limitations
- **Miter uses only the first primary output** (`primary_outputs[0]`) for now.
- The Verilog parser is tailored to these ISCAS-style gate-instance netlists (not general RTL).
- Next major milestone is adding the **LLM guidance layer** and comparing solver metrics against this baseline.
